02

개념설명

⏱ 20분▾
🎤 소리 센서
측정 범위: 0 ~ 255
뒷면 MICROPHONE 칩에 내장
큰 소리 감지 시 앞면 마이크 LED 점등
기준값으로 큰 소리 여부 판별
🔊 스피커
다양한 주파수로 음 출력
멜로디 블록 → 곡 재생
음표 블록 → 개별 음 재생
bpm 설정으로 속도 조절

알고리즘 설계 (4단계)

➊ 시작 → '작동 시작' 변수 0 초기화 + 소리 기준값 250 설정 + 시작 멜로디 재생
➋ 큰 소리 감지 → '작동 시작' 변수에 1 저장
➌ '작동 시작' = 1 이면 → 뛰어오르는·뛰어내리는 멜로디 순서대로 출력
➍ '작동 시작' = 1 이면 → LED에 춤 추는 애니메이션 출력

MakeCode 블록 예시

🟢 시작하면
작동시작 ← 0
소리 기준값 250으로 설정
🎵 전원켜는 멜로디 재생
🔊 큰 소리 감지 시
작동시작 ← 1
🔁 무한 반복
만약 작동시작 = 1
🎵 뛰어오르는 멜로디 (1번)
🎵 뛰어내리는 멜로디 (1번)

모둠 활동 | pp.31 ▾
💡 스피커와 소리 센서를 활용한 세 가지 수행 활동입니다. (교과서 p.31 해보기)
📋 활동 1 — 학교 종 노래 재생 프로그램 (pp.31)
🎵 학교 종 악보 (솔=G4, 라=A4, 미=E4, 도=C4, 레=D4)
학-교-종-이-땡-땡-땡: 솔솔라라솔솔미
어-서-모-이-자: 솔솔미미레
선-생-님-이-우-리-를: 솔솔라라솔솔미
기-다-리-신-다: 솔미레미도
📋 활동 2 — 노래에 어울리는 애니메이션 구상
학교 종 노래와 어울리는 LED 애니메이션을 구상하고, 각 장면을 5×5 LED 격자에 그려보세요.
LED 격자를 클릭하여 켜고 끄세요
📋 활동 3 — 두 모둠 박수 소리 누적 비교 장치 (pp.31)
조건:
① A 버튼 → 5초 동안 소리 크기를 그래프로 보여주며 값 누적 저장
② B 버튼 → 5초 동안 소리 크기를 그래프로 보여주며 값 누적 저장
③ 로고 터치 → A와 B 누적값 비교 후 큰 쪽 문자('A' or 'B') 출력

확인하기

⏱ 5분▾
소리 센서0~255 소리 크기. 기준값으로 큰 소리 판별.
작동 시작 변수0=대기, 1=작동. 큰 소리 시 1로 변경.
무한 반복 병렬멜로디·애니메이션을 별도 반복으로 동시 실행.
✏️ 단답형: 소리에 반응하는 애니메이션 장치에서 큰 소리가 감지되었을 때 1, 대기 상태일 때 0을 저장하는 것을  (이)라고 한다.
