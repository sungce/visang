UNIT Ⅰ — 2-04

📡 이심전심 게임 장치 만들기

두 명이 각자의 마이크로비트로 동시에 같은 것을 선택하면 이기는 텔레파시 게임! 마이크로비트 무선(Radio) 통신으로 두 기기가 데이터를 주고받는 네트워크 피지컬 컴퓨팅 시스템을 만들어봅시다.

2차시 · 90분
교과서 33~38쪽
마이크로비트 2대 필요
01
학습 목표
2차시 · 이 소단원을 마치면 할 수 있어요
▶
[9정01-04] 2022 개정 성취기준
  • 무선 통신(Radio)의 개념과 채널·그룹의 역할을 이해하고, 마이크로비트 Radio 모듈의 동작 원리를 설명할 수 있다.
  • radio.send()와 radio.receive()를 사용해 두 마이크로비트 간에 데이터를 주고받는 프로그램을 작성할 수 있다.
  • 게임 진행 규칙을 통신 프로토콜(메시지 형식·순서)로 설계하고 문서화할 수 있다.
  • 이심전심 게임에서 두 플레이어의 선택을 비교하여 승패를 판정하는 알고리즘을 설계하고 구현할 수 있다.

🆕이번 단원의 핵심 개념

지금까지는 마이크로비트 1대로 동작했습니다. 이번 단원에서는 처음으로 2대가 무선으로 통신합니다. 네트워킹(IoT)의 기초입니다.

🔗이전 단원과의 연계

입력(버튼) → 처리(난수·비교) → 출력(LED)의 기본 구조는 같지만, 처리 단계에 Radio 송수신이 추가됩니다.

🌐실세계 연결

스마트폰 Bluetooth, Wi-Fi, NFC 카드, 리모컨 — 모두 무선 통신의 원리를 사용합니다. 마이크로비트 Radio는 2.4GHz ISM 대역을 사용합니다.

03
마이크로비트 Radio 모듈 사용법
핵심 함수 4가지 — 클릭해 코드와 설명을 확인하세요
▶
💡 마이크로비트 Radio 모듈은 같은 채널(channel)·그룹(group)으로 설정된 기기끼리만 통신합니다. 수업에서 다른 팀과 혼선을 막으려면 팀마다 다른 그룹 번호를 사용하세요 (0~255).
▸ Radio 통신 흐름 예시
# 기기 A (송신)
import radio
radio.on() # 라디오 켜기
radio.config(group=5) # 그룹 5로 설정
radio.send("scissors") # "scissors" 전송
# 기기 B (수신)
msg = radio.receive() # 메시지 수신 대기
if msg == "scissors": # 수신값 확인
display.show(Image.NO) # "가위" 아이콘

🎮 선택지 설계 — 3가지 카테고리 중 하나를 선택합니다

✅이심전심 성공 (Win)

두 플레이어가 동시에 같은 것을 선택했을 때! 텔레파시 성공! LED에 하트♥를 표시하고 효과음을 냅니다. 선택지 3개 중 1개가 같을 확률 = 1/3 (약 33%).

❌이심전심 실패 (Fail)

두 플레이어가 서로 다른 것을 선택했을 때. LED에 X(틀림) 표시 후 양쪽의 선택을 공개합니다. 다시 도전하여 텔레파시를 맞춰보세요!

🎯실제 마이크로비트에서는

두 사람이 각자의 마이크로비트를 들고, 동시에 버튼을 누릅니다. 누른 버튼(A=첫 번째, B=두 번째, A+B=세 번째)이 선택지이고, radio.send()로 상대방에게 전송됩니다.

📡동시 전송의 비밀

실제로는 두 기기가 메시지를 동시에 보내고 동시에 받습니다. 먼저 상대방 메시지를 받으면 내 선택과 비교해 판정합니다. 그래서 양쪽 모두 같은 판정 결과를 보게 됩니다.

08
코딩 구현 — MicroPython
탭으로 플레이어 1 코드 / 플레이어 2 코드 / 4단원 종합 비교 확인
▶
📊 피지컬 컴퓨팅 프로그래밍 4단원 종합 비교
① 운동 카운터 (센서→카운트)
# 가속도 → 카운트 accel = strength() if accel > 700: count += 1 # 단방향: 센서만 사용
② 온도 알림 (다중 조건)
# 3분기 if~elif~else if temp < 18: display.show(COLD) elif temp <= 22: display.show(OK) else: display.show(HOT)
③ 소리 애니메이션 (Image)
# 4분기 + 커스텀 Image snd = sound_level() if snd < 30: clear() elif snd < 100: show(SMALL) elif snd < 180: show(MED) else: show(BIG)
④ 이심전심 (🆕 무선 통신)
# 무선 송수신 추가! radio.send(my_choice) msg = radio.receive() if msg == my_choice: show(Image.HEART) else: show(Image.NO)
🔑 핵심 발전 포인트: 단원 1→4로 진행하면서 처리 방식이 단일 조건 → 다중 조건 → 커스텀 패턴 → 무선 통신으로 발전합니다. 이번 단원에서 처음으로 두 기기 간 데이터 교환이 등장합니다.
