컴퓨터는 어떻게 자료를 이해할까?

컴퓨터는 0과 1, 두 가지 신호만 사용합니다.
이 두 숫자(비트)의 조합으로 문자·숫자·소리·이미지·영상을 모두 표현합니다.

비트(Bit): 컴퓨터가 다루는 가장 작은 데이터 단위 (0 또는 1)
바이트(Byte): 8개의 비트 = 1바이트 (256가지 정보 표현 가능)
디지털화: 실세계의 모든 정보를 0과 1로 변환하는 과정

알파벳·한글·숫자 등 모든 글자는
코드표(ASCII, 유니코드)를 이용해 이진수로 저장됩니다.

문자 표현

컴퓨터는 글자를 숫자 코드로 변환하여 저장합니다.
알파벳은 ASCII, 전 세계 문자는 유니코드(Unicode)를 사용합니다.

ASCII 코드란?

ASCII(아스키): 1963년 미국에서 만든 최초의 표준 문자 코드
영어 대소문자·숫자·특수문자 128가지를 7비트로 표현
예) A = 65, a = 97, 0 = 48

문자 10진수 2진수
A 65 01000001
B 66 01000010
a 97 01100001
0 48 00110000
! 33 00100001

유니코드(Unicode)란?

ASCII는 영어만 표현 가능 → 세계 모든 언어를 통일된 코드로!
유니코드: 14만 자 이상 수용, 한글·한자·아랍어·이모지 모두 포함
한글 '가' = U+AC00 = 44032 = 1010110000000000

UTF-8 UTF-16 한글 11,172자 이모지 포함

💡 스마트폰에서 한글 문자를 보낼 수 있는 이유가 바로 유니코드 때문입니다!

🧪 체험: 문자 → 이진수 변환기

변환 결과가 여기에 표시됩니다.

💡 문자 표현 핵심 정리

1

문자에 번호 부여

각 문자마다 고유한 숫자 코드를 정해 놓은 약속표를 만든다 (코드표).

2

10진수 → 2진수 변환

코드 번호를 이진수로 변환하면 컴퓨터가 읽을 수 있는 형태가 된다.

3

저장 및 전송

이진수로 변환된 값을 메모리에 저장하거나 네트워크로 전송한다.

🔢

수치 표현

컴퓨터는 10진수를 2진수(이진법)로 변환하여 처리합니다.
일상에서 쓰는 숫자가 컴퓨터 안에서 어떻게 바뀌는지 알아봅시다.

10진수 vs 2진수

10진수: 0~9 숫자 10개 사용, 자리가 넘으면 10을 올림
2진수: 0과 1만 사용, 자리가 넘으면 2를 올림
예) 10진수 13 = 2진수 1101

10진수 2진수 비트 수
0 0 1비트
1 1 1비트
5 101 3비트
10 1010 4비트
255 11111111 8비트

변환 방법: 나눗셈 반복

13을 2진수로 바꾸는 방법:
13 ÷ 2 = 6 나머지 1
6 ÷ 2 = 3 나머지 0
3 ÷ 2 = 1 나머지 1
1 ÷ 2 = 0 나머지 1
↑ 나머지를 아래부터 읽으면: 1101

2진수 1101 검증: 1×8 + 1×4 + 0×2 + 1×1 = 13 ✓

소리 표현

소리는 공기의 진동(아날로그 신호)입니다.
표본화 → 양자화 → 부호화 3단계로 디지털 데이터로 변환합니다.

아날로그 → 디지털 변환 과정 (ADC)

1

표본화 (Sampling) — "얼마나 자주 측정할까?"

연속적인 음파를 일정 시간 간격으로 잘라서 측정합니다.
샘플링률: 1초에 측정하는 횟수 (Hz)
예) CD 음질: 44,100Hz → 1초에 44,100번 측정

2

양자화 (Quantization) — "진폭 값을 숫자로!"

측정한 음파의 높이(진폭)를 정해진 단계의 숫자 값으로 근사합니다.
비트 심도: 16비트 → 65,536단계로 표현

3

부호화 (Encoding) — "이진수로 저장!"

양자화된 숫자 값을 이진수로 변환하여 파일에 저장합니다.
예) 진폭 값 1024 → 0000010000000000

🎵 소리 파형 시각화 체험

낮은 음 (저주파) 5 Hz 높은 음 (고주파)

↑ 파형: 곡선이 촘촘할수록 고음, 파형의 높이(진폭)가 클수록 큰 소리

음질과 파일 크기

파일 크기 = 샘플링률 × 비트심도 × 채널 수 × 시간(초)

CD 음질 1분:
44,100 × 16 × 2 × 60 = 약 10MB

실생활 예시

품질 샘플링률 용도
전화 8,000 Hz 통화
CD 44,100 Hz 음악
DVD 48,000 Hz 영화
고음질 96,000 Hz 스튜디오

이미지 표현

이미지는 픽셀(화소)이라는 작은 점들의 집합입니다.
각 픽셀의 색상은 RGB(빨강·초록·파랑) 세 값의 조합으로 표현됩니다.

픽셀(Pixel)이란?

픽셀(Pixel) = Picture + Element (그림 요소)
화면을 이루는 가장 작은 점 하나
1920×1080 Full HD = 약 207만 개의 픽셀
각 픽셀 = RGB 세 값 × 각 8비트 = 24비트

💡 스마트폰 화면을 확대해보면 작은 점들이 보입니다 — 그것이 픽셀!

RGB 색상 모델

빛의 3원색을 조합하여 색 표현
R(빨강) 0~255 + G(초록) 0~255 + B(파랑) 0~255
흰색: (255, 255, 255)
검정: (0, 0, 0)
빨강: (255, 0, 0)

영상 표현

영상은 정지 이미지(프레임)를 빠르게 연속으로 보여주는 것입니다.
인간의 눈은 1초에 약 24장 이상의 그림이 바뀌면 움직이는 것처럼 인식합니다.

프레임(Frame)이란?

영상에서 사용하는 하나의 정지 이미지
FPS(Frames Per Second): 초당 프레임 수
24 FPS → 영화
30 FPS → TV 방송
60 FPS → 게임, 스포츠 중계
120 FPS → 고성능 게이밍

영상 데이터 크기 계산

영상 크기 = 프레임 크기 × FPS × 시간

1080p, 30FPS, 1분 영상:
6.2MB × 30 × 60 = 약 11.2 GB
(압축 전 기준, 실제는 H.264 등으로 1/100 압축)

🎞 프레임 연속 재생 체험

1 FPS (끊김) 60 FPS (부드러움)
Frame: 0

FPS를 낮추면 끊기는 느낌이 납니다 — 실제 영상도 같은 원리입니다!

영상 압축 기술

I

I-Frame (키프레임) — 완전한 이미지

기준이 되는 전체 이미지를 저장합니다. 용량이 큽니다.

P

P-Frame (예측 프레임) — 변화된 부분만

이전 프레임과 달라진 부분만 저장합니다. 용량을 크게 줄입니다.

B

B-Frame (양방향 프레임) — 앞뒤 참조

앞뒤 프레임을 모두 참조하여 가장 효율적으로 압축합니다.

H.264 H.265 / HEVC AV1 MP4 MKV
